**2. Create `web_frontend/src/utils/extractCharacterName.ts`:**

Pure function for extracting character name from ai-instructions text. Per discretion recommendation in RESEARCH.md:

```typescript
/**
 * Extract character name from roleplay ai-instructions text.
 *
 * Looks for common patterns:
 * - "You are [Name]" at start
 * - "Character: [Name]" or "Name: [Name]" on any line
 * Falls back to "Character" if no match found.
 */
export function extractCharacterName(aiInstructions: string): string {
  // Match "You are [Name]" at start of instructions
  // Name: starts with uppercase, 1-30 chars, ends at period, comma, or newline
  const youAreMatch = aiInstructions.match(
    /^You are\s+([A-Z][a-zA-Z\s]{1,30}?)[\.,\n]/
  );
  if (youAreMatch) return youAreMatch[1].trim();

  // Match "Character: [Name]" or "Name: [Name]" on any line
  const labelMatch = aiInstructions.match(
    /^(?:Character|Name):?\s+([A-Z][a-zA-Z\s]{1,30}?)[\.,\n]/m
  );
  if (labelMatch) return labelMatch[1].trim();

  return "Character"; // Fallback
}
```

**1. Create `web_frontend/src/hooks/useRoleplaySession.ts`:**

This hook manages the full roleplay session lifecycle — isolated from Module.tsx's shared chat state (per Pitfall 2 in RESEARCH.md).

```typescript
export interface UseRoleplaySessionReturn {
  messages: Array<{ role: string; content: string }>;
  pendingMessage: { content: string; status: "sending" | "failed" } | null;
  streamingContent: string;
  isLoading: boolean;
  isCompleted: boolean;
  sessionId: number | null;
  /** Full assistant response text from the most recent streaming response (for TTS) */
  lastAssistantResponse: string | null;
  sendMessage: (content: string) => void;
  complete: () => Promise<void>;
  retry: () => Promise<void>;
}

export function useRoleplaySession(
  moduleSlug: string,
  roleplayId: string,
  aiInstructions: string,
  scenarioContent: string | undefined,
  openingMessage: string | undefined,
): UseRoleplaySessionReturn
```

**State:**
- `messages`: Array of `{role, content}` — loaded from history on mount
- `pendingMessage`: Current user message being sent (null when idle)
- `streamingContent`: Accumulating assistant response text during streaming
- `isLoading`: True while streaming
- `isCompleted`: True if session has completedAt
- `sessionId`: From history response
- `lastAssistantResponse`: Set to the full assistant text when streaming completes (reset to null when next message starts)

**Behavior:**
- **On mount:** Call `getRoleplayHistory(moduleSlug, roleplayId)` to load existing session. If messages are empty and openingMessage is provided, trigger initial load by calling `sendRoleplayMessage` with empty message (this causes the backend to insert and return the opening message).
- **sendMessage(content):** Set pendingMessage, call `sendRoleplayMessage()`, accumulate text events into streamingContent. On "done" event: append complete message to messages, clear pendingMessage and streamingContent, set lastAssistantResponse. On error: set pendingMessage status to "failed".
- **complete():** Call `completeRoleplay(sessionId)`, set isCompleted to true.
- **retry():** Call `retryRoleplay(sessionId, openingMessage)`. Reset all state (messages, pendingMessage, streamingContent). Set sessionId to new session's ID. If openingMessage exists, trigger initial load like on mount.

**Important:** Use an AbortController ref to cancel in-flight SSE streams on unmount or retry. Store the async generator reference to allow cleanup. Follow the pattern from the Prompt Lab's `streamAbortRef`.

**3. Create `web_frontend/src/hooks/useRoleplayTTS.ts`:**

Coordinates sending completed assistant text to the TTS WebSocket for audio playback. Uses the **buffered TTS** approach per RESEARCH.md recommendation (send full text after LLM response completes, not streaming TTS).

```typescript
export interface UseRoleplayTTSReturn {
  /** Call after LLM response completes to speak the text */
  speakText: (text: string) => void;
  /** Stop playback and close WebSocket */
  stop: () => void;
  /** Whether audio is currently playing */
  isPlaying: boolean;
}

export function useRoleplayTTS(ttsEnabled: boolean): UseRoleplayTTSReturn
```

**Behavior:**
- Uses `useAudioPlayback()` internally for gapless chunk playback
- `speakText(text)`:
  1. If !ttsEnabled or !text.trim(), return early
  2. Call `audioPlayback.resume()` (ensures AudioContext is active after user gesture)
  3. Open WebSocket to `/ws/tts` (use `location.protocol === "https:" ? "wss:" : "ws:"` + `location.host`)
  4. On open: send `JSON.stringify({ text, voice: "Ashley" })`
  5. On message: if binary Blob, convert to ArrayBuffer and call `audioPlayback.playChunk(bytes)`
  6. On JSON `{"done": true}`: close WebSocket
  7. On JSON `{"error": "..."}`: log error, close WebSocket
  8. Store WebSocket ref for cleanup
- `stop()`: Call `audioPlayback.stop()`, close active WebSocket if any
- Cleanup on unmount: close WebSocket and stop audio
- Handle the AudioContext autoplay policy per Pitfall 3 in RESEARCH.md — the `speakText` call should happen in a chain from a user gesture (message send button click)
