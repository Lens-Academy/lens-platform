Create `web_api/routes/roleplay.py` with a dedicated roleplay chat API, following the same patterns as `web_api/routes/module.py` but with key differences:

**Router setup:** `APIRouter(prefix="/api/chat", tags=["roleplay"])`

**Request/Response models:**
```python
class RoleplayChatRequest(BaseModel):
    module_slug: str          # For module_id lookup
    roleplay_id: str          # UUID string from content
    message: str              # User message (empty string for initial load)
    ai_instructions: str      # Character behavior from content
    scenario_content: str | None = None  # Briefing text
    opening_message: str | None = None   # Optional first AI message

class RoleplayHistoryResponse(BaseModel):
    sessionId: int
    messages: list[dict]
    completedAt: str | None = None

class RetryRequest(BaseModel):
    opening_message: str | None = None
```

**Endpoints:**

1. `POST /api/chat/roleplay` — SSE stream for roleplay chat
   - Auth: `get_user_or_anonymous` (same as module chat)
   - Load module via `load_flattened_module(request.module_slug)` to get `module.content_id`
   - Call `get_or_create_chat_session()` with `module_id=module.content_id, roleplay_id=UUID(request.roleplay_id)`
   - **Opening message handling:** If session is new (empty messages) AND `request.opening_message` is set:
     - Save opening message via `add_chat_message(role="assistant", content=request.opening_message)`
     - Yield it as SSE: `data: {"type":"text","content":"<opening_message>"}\n\n` then `data: {"type":"done"}\n\n`
     - Return immediately (no LLM call)
   - **Normal message flow:** Save user message, build system prompt via `build_roleplay_prompt()`, build LLM message history from existing messages + new user message, stream via `stream_chat()` with `tools=None` (no stage transitions)
   - Save assistant response after streaming completes
   - **Check completed_at:** If session has `completed_at` set, return error event: `data: {"type":"error","message":"Session completed"}\n\n` and `data: {"type":"done"}\n\n`
   - Error handling: same pattern as module.py (try/except around stream_chat, yield error event, log to Sentry)
   - Response headers: `text/event-stream`, `Cache-Control: no-cache`, `X-Accel-Buffering: no`

2. `GET /api/chat/roleplay/{roleplay_id}/history` — Get roleplay chat history
   - Auth: `get_user_or_anonymous`
   - Query param: `module_slug` (required)
   - Load module to get content_id
   - Call `get_or_create_chat_session()` with roleplay_id
   - Return `RoleplayHistoryResponse` with `completedAt` as ISO string if set

3. `POST /api/chat/roleplay/{session_id}/complete` — Complete a roleplay session
   - Auth: `get_user_or_anonymous`
   - Get session by ID, verify ownership (user_id or anonymous_token matches)
   - Call `complete_chat_session(conn, session_id=session_id)`
   - Return `{"status": "completed"}`

4. `POST /api/chat/roleplay/{session_id}/retry` — Retry with fresh session
   - Auth: `get_user_or_anonymous`
   - Body: `RetryRequest` with optional `opening_message`
   - Get session by ID, verify ownership
   - Archive old session via `archive_chat_session()`
   - Create new session via `get_or_create_chat_session()` using same module_id and roleplay_id from old session
   - If `opening_message` provided, insert as assistant message in new session
   - Return `{"sessionId": new_session["session_id"]}`

- Opening message logic for new sessions
