**core/tts/inworld_ws.py** -- Persistent WebSocket client:
- `InworldTTSClient` class following the pattern from 09-RESEARCH.md Pattern 1.
- Uses `websockets.asyncio.client.connect` (websockets 16.0 already installed).
- `_ensure_connected()` method: connects/reconnects if needed, uses `Basic {api_key}` auth header.
- `synthesize(text_chunks: AsyncIterator[str], config: TTSConfig | None = None, context_id: str | None = None) -> AsyncIterator[bytes]`:
  1. Ensure connected
  2. Send `create` message with config values and a unique context_id (use uuid4 if not provided)
  3. Wait for `contextCreated` response
  4. Start an asyncio task that sends each text chunk via `send_text` messages, then sends `flush_context`
  5. Yield decoded audio bytes from `audioChunk` messages until `flushCompleted`
  6. Send `close_context` to clean up
  7. Use try/finally to ensure the send task is awaited and context is closed
- `close()` method: closes WebSocket connection.
- `_keepalive_loop()` method: sends empty text every 60 seconds to prevent timeout (started as background task when connected).
- Use `asyncio.Lock` to serialize `_ensure_connected` (but NOT the entire synthesize -- multiple contexts can run on one connection, just serialize connection setup).
- Handle `websockets.exceptions.ConnectionClosed` in synthesize by reconnecting and retrying once.
- Logging via `logging.getLogger(__name__)` for connection events, errors.

**core/tts/__init__.py** -- Public API:
- Import and re-export `InworldTTSClient`, `TTSConfig`, `is_tts_available`, `get_api_key`
- Module-level singleton: `_client: InworldTTSClient | None = None`
- `get_tts_client() -> InworldTTSClient`: lazy-init singleton, returns existing or creates new.
- `close_tts_client()`: closes singleton if exists, sets to None (for shutdown cleanup).

**web_api/routes/tts_stream.py** -- FastAPI WebSocket endpoint:
- `router = APIRouter(tags=["tts"])`
- `@router.websocket("/ws/tts")` endpoint following Pattern 2 from research.

Protocol:
1. Client connects to `/ws/tts`
2. Client sends JSON: `{"text": "full text to synthesize", "voice": "Ashley"}` (for Phase 9 test harness, text is the complete response -- Phase 10 will add streaming mode where text comes from LLM)
3. Server sends binary WebSocket frames (MP3 audio chunks) as they arrive from Inworld
4. Server sends JSON `{"done": true}` when all audio has been sent
5. Server sends JSON `{"error": "message"}` if TTS fails

Implementation:
- Accept WebSocket connection
- Receive JSON request with `text` and optional `voice` fields
- Check `is_tts_available()` -- if not, send `{"error": "TTS not configured (INWORLD_API_KEY not set)"}` and close
- Create an async iterator that yields text (for Phase 9: yield the full text as a single chunk -- simple and correct for test harness. Phase 10 will provide an actual token stream)
- Call `get_tts_client().synthesize(text_iter, config)` where config uses the requested voice
- Forward each audio chunk as `websocket.send_bytes(chunk)`
- Send `{"done": true}` JSON message on completion
- Wrap in try/except for WebSocketDisconnect (client left), log and clean up
- Wrap in try/except for general exceptions, send error JSON before closing
