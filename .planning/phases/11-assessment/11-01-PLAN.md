**Part A: Create `core/roleplay_assessment.py`** mirroring `core/assessment.py` pattern.

The module needs:

1. `ROLEPLAY_ASSESSMENT_SYSTEM_PROMPT` -- base system prompt for scoring roleplay transcripts. Something like: "You are a rigorous educational assessor evaluating a student's performance in a roleplay scenario. Score the student's responses against the rubric based on the full conversation transcript. Measure actual understanding and skill demonstrated, not effort."

2. `ROLEPLAY_PROMPT_VERSION = "v1"` -- version tracker.

3. `_running_tasks: set[asyncio.Task]` -- same GC prevention pattern.

4. `format_transcript(messages: list[dict]) -> str` -- formats chat_sessions.messages for the prompt:
   - user messages as "Student: {content}"
   - assistant messages as "Character: {content}"
   - joined with double newlines

5. `_build_roleplay_scoring_prompt(*, transcript: str, assessment_instructions: str, scenario_context: str | None = None) -> tuple[str, list[dict]]` -- builds system prompt + messages list. Appends rubric and scenario context to system prompt.

6. `enqueue_roleplay_scoring(session_id: int, messages: list[dict], segment_snapshot: dict) -> None` -- fire-and-forget entry point. Creates asyncio task calling `_score_roleplay_session()`.

7. `_task_done(task)` -- same cleanup callback as assessment.py.

8. `_score_roleplay_session(session_id: int, messages: list[dict], segment_snapshot: dict) -> None`:
   - Extract `assessment_instructions` from segment_snapshot (try both camelCase `assessmentInstructions` and kebab-case `assessment-instructions`)
   - Extract `scenario_context` from snapshot's `content` field
   - Format transcript using `format_transcript()`
   - Build prompt using `_build_roleplay_scoring_prompt()`
   - Call `complete()` from `core/modules/llm.py` with `response_format=SCORE_SCHEMA` (import SCORE_SCHEMA from `core/assessment`)
   - Parse JSON response
   - Write to `roleplay_assessments` table using `get_transaction()`
   - Log the result

Use `SCORING_PROVIDER` from `core/assessment` (import it, don't duplicate).

**Part B: Update `web_api/routes/roleplay.py`**

1. **Populate segment_snapshot in the SSE endpoint.** In `roleplay_event_generator()`, when calling `get_or_create_chat_session()`, pass `segment_snapshot` built from the request data:
   ```python
   segment_snapshot={
       "content": request.scenario_content,
       "aiInstructions": request.ai_instructions,
       "openingMessage": request.opening_message,
       "assessmentInstructions": request.assessment_instructions,
   }
   ```
   This requires adding `assessment_instructions: str | None = None` to the `RoleplayChatRequest` model.

2. **Trigger scoring in `complete_roleplay()`.** After `complete_chat_session()` succeeds:
   - Re-fetch the session to get `messages` and `segment_snapshot`
   - Check if `segment_snapshot` has `assessmentInstructions` or `assessment-instructions`
   - If rubric exists, call `enqueue_roleplay_scoring(session_id, messages, segment_snapshot)`
   - Import `enqueue_roleplay_scoring` from `core.roleplay_assessment`

3. **Add score retrieval endpoint.** Create `GET /api/chat/roleplay/{session_id}/assessment`:
   - Takes session_id path param
   - Auth via `get_user_or_anonymous`
   - Queries `roleplay_assessments` table for the given session_id
   - Returns the most recent assessment's `score_data`, `model_id`, `prompt_version`, `created_at` or 404 if none exists
   - Ownership check: verify session belongs to caller via chat_sessions join
   - Add Pydantic response model `RoleplayAssessmentResponse` with fields: `score_data: dict`, `model_id: str | None`, `created_at: str`
