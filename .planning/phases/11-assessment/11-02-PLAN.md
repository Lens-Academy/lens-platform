**Part A: Create `TestRoleplayCard.tsx`**

A wrapper that renders a RoleplaySection inside a test section card with reveal/collapse behavior matching TestQuestionCard's visual pattern. Key differences from standalone RoleplaySection:
- Accepts `isActive`, `isCompleted`, `isRevealed` props (same as TestQuestionCard)
- Calls `onComplete()` callback when the roleplay session completes (via the `completeRoleplay` API or `useUnifiedRoleplay.markComplete`)
- Shows collapsed state when not active/revealed (just "Roleplay Exercise" label)
- Shows completion checkmark when completed

The component should:
1. Accept props: `segment: RoleplaySegment`, `moduleSlug: string`, `itemIndex: number`, `itemCount: number`, `isActive: boolean`, `isCompleted: boolean`, `isRevealed: boolean`, `onComplete: () => void`
2. When collapsed (`!isRevealed`): show a summary card with "Roleplay" label and locked icon
3. When revealed but not active: show collapsed card with segment.content preview
4. When active (`isActive && !isCompleted`): render the full RoleplaySection with an override so that when the user clicks "End Conversation", it calls both the roleplay completion API AND the `onComplete` callback
5. When completed: show checkmark and "Completed" status

For the active state, the simplest approach is to render a modified version of RoleplaySection that accepts an `onComplete` callback prop. This means updating RoleplaySection to accept an optional `onComplete?: () => void` prop. When provided, it calls `onComplete()` after `handleComplete` succeeds. This is the same pattern used by AnswerBox's `onComplete` prop for test sections.

Update `RoleplaySection.tsx` to accept optional `onComplete?: () => void` prop. In `handleComplete`, after `completeRoleplay(sessionId)` and `markComplete()`, call `onComplete?.()`.

**Part B: Refactor TestSection for unified assessable items**

The current TestSection extracts only `QuestionInfo[]`. Refactor to track a unified list of assessable items:

```typescript
type AssessableItem =
  | { type: "question"; segment: QuestionSegment; segmentIndex: number }
  | { type: "roleplay"; segment: RoleplaySegment; segmentIndex: number };
```

Changes to TestSection:
1. Replace `questions: QuestionInfo[]` useMemo with `assessableItems: AssessableItem[]` that filters for both `seg.type === "question"` and `seg.type === "roleplay"`.

2. Update `loadResponses` effect: For question items, load via `getResponses()` as before. For roleplay items, load via `getRoleplayHistory()` from `@/api/roleplay` -- if the session has `completedAt`, mark as completed. Use `Promise.allSettled` to handle mixed results.

3. Update item count: `assessableItems.length` instead of `questions.length` throughout.

4. Update Begin screen: show "X question(s) and Y roleplay(s)" (or just "X item(s)" if simpler).

5. Update render loop: Map over `assessableItems` instead of `questions`. For question items, render `TestQuestionCard`. For roleplay items, render `TestRoleplayCard`.

6. Update completion check: `completedItems.size === assessableItems.length`.

7. Update `handleQuestionComplete` to be `handleItemComplete(itemIndex: number)`.

8. Update feedback trigger: only include question answers (roleplay has its own feedback path). Filter to question items when building questionsAndAnswers for `onFeedbackTrigger`.
